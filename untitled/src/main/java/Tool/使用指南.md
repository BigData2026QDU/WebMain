# HibernateUtil 使用指南

## 简介

HibernateUtil 是一个全局静态工具类，封装了 Hibernate ORM 的常用操作，简化数据库访问代码。

## 前置条件

1. 确保 `hibernate.cfg.xml` 配置文件存在于 `src/main/resources/` 目录
2. 在配置文件中注册所有实体类
3. 数据库连接信息配置正确

## 基础用法

### 1. 保存数据

```java
import Tool.HibernateUtil;
import org.example.User;

// 创建并保存新用户
User user = new User("张三", "zhangsan@example.com");
HibernateUtil.save(user);
```

### 2. 查询数据

```java
// 根据 ID 查询
User user = HibernateUtil.findById(User.class, 1L);

// 查询所有用户
List<User> allUsers = HibernateUtil.findAll(User.class);
```

### 3. 更新数据

```java
// 查询用户
User user = HibernateUtil.findById(User.class, 1L);

// 修改属性
user.setName("李四");
user.setEmail("lisi@example.com");

// 保存更新
HibernateUtil.update(user);
```

### 4. 删除数据

```java
User user = HibernateUtil.findById(User.class, 1L);
HibernateUtil.delete(user);
```

## 高级用法

### 1. 自定义事务操作

```java
HibernateUtil.executeInTransaction(session -> {
    // 批量保存
    for (int i = 0; i < 100; i++) {
        User user = new User("用户" + i, "user" + i + "@example.com");
        session.persist(user);

        // 每 20 条刷新一次
        if (i % 20 == 0) {
            session.flush();
            session.clear();
        }
    }
});
```

### 2. 复杂查询

```java
// 使用 executeQuery 执行自定义查询
List<User> users = HibernateUtil.executeQuery(session ->
    session.createQuery("FROM User WHERE name LIKE :pattern", User.class)
           .setParameter("pattern", "%张%")
           .setMaxResults(10)
           .list()
);
```

### 3. HQL 查询

```java
// 条件查询
List<User> users = HibernateUtil.executeHQL(
    "FROM User WHERE email = ?0 AND name LIKE ?1",
    User.class,
    "test@example.com",
    "%张%"
);

// 统计查询
List<Long> count = HibernateUtil.executeHQL(
    "SELECT COUNT(u) FROM User u WHERE u.name LIKE ?0",
    Long.class,
    "%张%"
);
```

### 4. 批量更新/删除

```java
// 批量更新
int updated = HibernateUtil.executeUpdate(
    "UPDATE User SET name = ?0 WHERE id > ?1",
    "新名字",
    100L
);

// 批量删除
int deleted = HibernateUtil.executeUpdate(
    "DELETE FROM User WHERE id < ?0",
    10L
);
```

## 完整示例

```java
package org.example;

import Tool.HibernateUtil;
import java.util.List;

public class UserService {

    // 创建用户
    public void createUser(String name, String email) {
        User user = new User(name, email);
        HibernateUtil.save(user);
    }

    // 获取所有用户
    public List<User> getAllUsers() {
        return HibernateUtil.findAll(User.class);
    }

    // 根据邮箱查找用户
    public User findByEmail(String email) {
        List<User> users = HibernateUtil.executeHQL(
            "FROM User WHERE email = ?0",
            User.class,
            email
        );
        return users.isEmpty() ? null : users.get(0);
    }

    // 更新用户信息
    public void updateUser(Long id, String newName) {
        User user = HibernateUtil.findById(User.class, id);
        if (user != null) {
            user.setName(newName);
            HibernateUtil.update(user);
        }
    }

    // 删除用户
    public void deleteUser(Long id) {
        User user = HibernateUtil.findById(User.class, id);
        if (user != null) {
            HibernateUtil.delete(user);
        }
    }

    // 批量操作示例
    public void batchCreateUsers(List<String> names) {
        HibernateUtil.executeInTransaction(session -> {
            for (int i = 0; i < names.size(); i++) {
                User user = new User(names.get(i), "user" + i + "@example.com");
                session.persist(user);

                if (i % 20 == 0) {
                    session.flush();
                    session.clear();
                }
            }
        });
    }
}
```

## 注意事项

1. **事务管理**：所有写操作（save/update/delete）都自动包含事务管理
2. **异常处理**：所有方法在失败时会抛出 RuntimeException，需要在调用处捕获
3. **资源释放**：Session 会自动关闭，无需手动管理
4. **线程安全**：SessionFactory 是单例且线程安全的
5. **应用关闭**：在应用关闭时调用 `HibernateUtil.shutdown()` 释放资源

## 最佳实践

1. **使用 executeInTransaction 进行批量操作**
   ```java
   HibernateUtil.executeInTransaction(session -> {
       // 批量操作代码
   });
   ```

2. **大批量操作时定期刷新**
   ```java
   if (i % 20 == 0) {
       session.flush();
       session.clear();
   }
   ```

3. **查询优化**
   ```java
   // 使用 setMaxResults 限制结果数量
   query.setMaxResults(100);

   // 使用 JOIN FETCH 避免 N+1 问题
   "FROM User u JOIN FETCH u.orders WHERE u.id = ?0"
   ```

4. **异常处理**
   ```java
   try {
       HibernateUtil.save(user);
   } catch (RuntimeException e) {
       // 处理异常
       logger.error("保存用户失败", e);
   }
   ```

## 常见问题

**Q: 如何添加新的实体类？**
A: 在 `hibernate.cfg.xml` 中添加 `<mapping class="包名.类名"/>`

**Q: 事务失败会自动回滚吗？**
A: 是的，所有事务操作失败时会自动回滚

**Q: 可以嵌套事务吗？**
A: 不建议嵌套事务，应该在最外层使用 executeInTransaction

**Q: 如何执行原生 SQL？**
A: 使用 executeQuery 配合 session.createNativeQuery()
```java
HibernateUtil.executeQuery(session ->
    session.createNativeQuery("SELECT * FROM users", User.class).list()
);
```
